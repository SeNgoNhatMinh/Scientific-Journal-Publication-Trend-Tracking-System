# 業務要件一覧

| id | category | requirement | priority | description |
| --- | --- | --- | --- | --- |
| BR-001 | データ収集・管理（Research Corpus） | 外部学術データベース連携 | 高 | OpenAlex、Semantic Scholar、arXiv等の外部学術ソースからメタデータ（タイトル、著者、抄録、発行年等）を自動取得し、システム内に取り込む仕組みを構築する。 |
| BR-002 | データ収集・管理（Research Corpus） | データ正規化処理 | 高 | 異なるソースから取得したデータの形式（項目名やフォーマット）を統合し、分析可能な共通スキーマに変換して格納する。 |
| BR-003 | データ収集・管理（Research Corpus） | 重複データ排除（デデュプリケーション） | 中 | 同一の論文が異なるソースから重複して収集された場合、DOIやタイトル等の指標を用いて同一性を判定し、レコードを統合・整理する。 |
| BR-004 | データ収集・管理（Research Corpus） | データ自動更新スケジューリング | 高 | 最新の学術論文情報を定期的に自動収集し、Research Corpusを常に最新の状態に保つタスクを実行する。※要確認：更新頻度（日次/週次など）については未定義 |
| BR-005 | データ収集・管理（Research Corpus） | データ品質管理（クリーニング） | 中 | 収集したデータの中に含まれる欠損値や不整合な情報を検出し、補完または除外する処理を行う。 |
| BR-006 | データ収集・管理（Research Corpus） | データ保持ポリシーの管理 | 低 | システムで管理する論文データの保持期間やアーカイブ条件を規定し、運用ルールを管理する。※要確認：ライフサイクル管理のルールは未定義 |
| BR-007 | データ収集・管理（Research Corpus） | ソース接続監視 | 中 | 外部APIやデータソースへの接続状態を監視し、収集エラーが発生した際に管理者に通知する。※要確認：通知手段やエスカレーションフローは未定義 |
| BR-008 | データ収集・管理（Research Corpus） | データソース追加管理 | 低 | 将来的に新たな学術ソースやデータベースを追加するための拡張性を持ったデータ取り込み設定を行う。※要確認：追加対象ソースの優先順位は未定義 |
| BR-009 | 検索エンジン | 基本キーワード検索機能 | 高 | ユーザーが入力したキーワードに基づき、Research Corpus内の論文タイトル、要約、またはキーワードから該当する論文を検索し一覧表示する。 |
| BR-010 | 検索エンジン | 属性ベースの絞り込み機能 | 高 | 検索結果に対して、著者名、出版年、研究分野、出版元ソースなどの属性情報を指定してフィルタリングを行う。 |
| BR-011 | 検索エンジン | 高度な検索条件の組み合わせ | 中 | 複数の検索条件（論理演算子AND/OR/NOT等）を組み合わせた複雑なクエリによる検索を可能にする。※要確認：UIでの実装レベルについては未定義。 |
| BR-012 | 検索エンジン | 検索結果のソート・並び替え | 中 | 検索結果を関連度順、発行年（昇順・降順）、被引用数などの指標に基づいて並び替える。 |
| BR-013 | 検索エンジン | メタデータ詳細表示 | 高 | 検索結果一覧から特定の論文を選択した際、DOIやフルテキストへのリンクを含む詳細な論文情報を表示する。 |
| BR-014 | 検索エンジン | 検索条件の保存機能 | 低 | 頻繁に使用する検索条件やフィルタ設定を保存し、次回以降のアクセス時に即座に実行できるようにする。※要確認：ユーザーアカウント紐付けの要件詳細は未定義。 |
| BR-015 | 分析エンジン | 研究トレンドの時系列分析 | 高 | 蓄積されたResearch Corpusから、特定の研究分野やキーワードに関する論文数を年度別または月別に集計し、トレンドの変化を数値化する業務プロセス。 |
| BR-016 | 分析エンジン | 研究分野の成長速度算出 | 高 | 各研究分野における一定期間の論文発表増加率を算出し、現在どの分野が急成長しているかを判定する業務プロセス。 |
| BR-017 | 分析エンジン | 関連キーワード・トピックの提示 | 中 | 選択した研究テーマや検索キーワードに基づき、論文内で共起頻度が高い関連単語や、近接領域の学術トピックを自動抽出する業務プロセス。 |
| BR-018 | 分析エンジン | Research Gap（研究空白域）の発見 | 高 | 主要分野間で論文発表数を比較し、学術的な重要性やポテンシャルが高いにもかかわらず、既存論文数が少ない領域（研究の空白地帯）を特定する業務プロセス。 |
| BR-019 | 分析エンジン | 研究テーマの潜在的な関連性マッピング | 中 | 異なる研究分野間での引用関係や著者グループの重なりを分析し、学際的な研究のつながりや広がりを特定する業務プロセス。※要確認：分析ロジックの定義詳細 |
| BR-020 | 分析エンジン | 学術データの正規化・標準化処理 | 高 | 収集した各ソース（OpenAlex, IEEE等）の形式が異なるメタデータを分析エンジンが処理可能な形式へ統合・正規化する業務プロセス。※要確認：表記揺れの吸収精度 |
| BR-021 | 分析エンジン | 分析レポートの自動生成と更新 | 中 | 定期的に実行される分析結果を、ユーザーが閲覧可能な形式（サマリーレポート）として自動的に生成・更新する業務プロセス。※要確認：レポート生成頻度 |
| BR-022 | 可視化・ダッシュボード | 研究トレンドの可視化 | 高 | 主要な学術トピックや技術分野の発表数推移を折れ線グラフなどで時系列表示し、成長速度や注目の変化を視覚的に把握できるようにする。 |
| BR-023 | 可視化・ダッシュボード | Research Gap（研究空白域）の可視化 | 高 | トピックごとの研究密度をヒートマップやバブルチャートを用いて表示し、論文数が少なく研究余地がある領域を識別可能にする。 |
| BR-024 | 可視化・ダッシュボード | AI分析結果のインターフェース表示 | 中 | 論文要約や推奨研究テーマなど、AIエンジンが生成した分析結果をダッシュボード上で分かりやすく提示する。 |
| BR-025 | 可視化・ダッシュボード | 統計レポートの出力 | 中 | 検索条件に基づいた学術データの統計量（論文数、主要著者、関連キーワードなど）をサマリーレポートとして表示する機能。※要確認：PDFやCSVでのエクスポート機能の有無 |
| BR-026 | 可視化・ダッシュボード | カスタマイズ可能なダッシュボード | 低 | ユーザーが関心のあるトピックや統計グラフを個別のダッシュボードへ配置し、自身の研究ニーズに合わせて表示項目を調整できるようにする。※要確認：UIのプリセット機能が必要かどうか |
| BR-027 | パーソナライズ・ライブラリ管理 | 文献の保存機能 | 高 | ユーザーが検索結果から特定の論文を選択し、個人のマイライブラリに保存できること。※要確認：保存時にタグ付けなどのメタデータ付与が必要か |
| BR-028 | パーソナライズ・ライブラリ管理 | コレクション（フォルダ）管理機能 | 中 | 保存した文献を研究テーマやプロジェクト単位で分類できるよう、フォルダやコレクションを作成・編集・削除できること。 |
| BR-029 | パーソナライズ・ライブラリ管理 | 興味トピックの追跡設定 | 高 | ユーザーが特定の研究分野やキーワードをフォロー対象として設定し、マイページからいつでも一覧確認できること。 |
| BR-030 | パーソナライズ・ライブラリ管理 | 更新通知機能 | 中 | 追跡中のトピックやキーワードに関連する新しい論文が追加された際、ユーザーへ通知を送信する機能。※要確認：通知手段（メール、アプリ内通知など） |
| BR-031 | パーソナライズ・ライブラリ管理 | ライブラリ内の文献検索 | 中 | 保存済みの文献に対して、ユーザーがキーワードや保存日時による絞り込み・検索ができること。 |
| BR-032 | パーソナライズ・ライブラリ管理 | 文献情報の共有・エクスポート | 低 | ライブラリに保存した文献リストを外部へ共有、または文献情報を書誌管理ソフト等で利用可能な形式で出力できること。※要確認：対応フォーマット |
| BR-033 | AI支援機能 | 論文自動要約の提供 | 高 | ユーザーが選択した学術論文の全文または要約文に対し、AIが主要な論点、研究手法、および結論を簡潔に抽出して提示する。※要確認：要約の文字数や出力形式の要件 |
| BR-034 | AI支援機能 | 関連資料の動的レコメンデーション | 高 | 閲覧中の論文やユーザーの検索履歴に基づき、内容的に関連性の高い論文、引用文献、または類似したトピックの最新記事を自動的に提示する。 |
| BR-035 | AI支援機能 | 専門用語解説機能 | 中 | 論文本文内の専門用語や難解なフレーズを選択した際、AIがその分野の文脈に基づいた簡潔な定義や解説をポップアップまたはサイドバーで表示する。 |
| BR-036 | AI支援機能 | 研究テーマ・方向性の提案 | 高 | ユーザーの関心領域やResearch Gapの分析結果を基に、新規性が高く将来性の期待できる研究テーマや探索すべき方向性をAIが推奨する。 |
| BR-037 | AI支援機能 | AI分析結果の根拠表示 | 中 | AIが要約や提案を行う際、その結論に至った根拠となる特定の論文データやデータソースへのリンクを併せて提示し、情報の妥当性を担保する。 |
| BR-038 | ユーザー・システム管理 | ユーザーアカウント管理 | 高 | 学生、教員、研究者、管理者がシステムを利用するための登録、認証、およびプロファイル情報を管理するプロセス。※要確認：認証方式（SSO連携の有無など） |
| BR-039 | ユーザー・システム管理 | 役割ベースのアクセス制御（RBAC） | 高 | ユーザーの属性に応じて、利用可能な機能範囲や管理権限を定義し制御するプロセス。管理者はシステム全体の管理を行い、一般ユーザーは個人ライブラリや分析機能を利用する。 |
| BR-040 | ユーザー・システム管理 | システム運用保守管理 | 中 | システム設定の変更、データベースの整合性維持、および各機能が正常に稼働しているかを確認するための管理プロセス。※要確認：保守ツールや管理コンソールの要件 |
| BR-041 | ユーザー・システム管理 | システム利用ログの収集と監視 | 中 | ユーザーの検索実行履歴、アクセス頻度、およびシステムエラーの発生状況を記録し、異常検知や改善のためのデータ分析に活用するプロセス。※要確認：ログの保存期間や監査要件 |
| BR-042 | ユーザー・システム管理 | ユーザーフィードバック収集プロセス | 低 | システム利用者が研究支援機能に対して改善要望やフィードバックを送付する仕組み。管理者側で集約・優先順位付けし、システム改善に反映させるプロセス。 |
