# 非機能要件一覧

| id | category | requirement | targetValue | priority |
| --- | --- | --- | --- | --- |
| NFR-001 | 性能 | 検索・分析レスポンス時間 | 検索結果の表示および分析結果のグラフ描画は3秒以内とする。※要確認：データセット規模による性能限界値 | 高 |
| NFR-002 | 可用性 | システム稼働率 | 99.9%（メンテナンス時間を除く） | 中 |
| NFR-003 | セキュリティ | 認証・認可 | OAuth2.0/OpenID Connectを用いた安全な認証基盤を構築する。※要確認：SSO連携の有無 | 高 |
| NFR-004 | 運用 | データ収集バッチ処理の監視 | 外部APIからのデータ収集失敗時に管理者にアラート通知を行う。※要確認：許容される収集遅延時間 | 中 |
| NFR-005 | 移行 | データソースの追加・変更対応 | 新しい学術データベースの追加が設定変更のみで可能な構成にする。※要確認：将来的なデータソースの追加計画 | 中 |
| NFR-006 | 拡張性 | スケーラビリティ | 研究者ユーザーの増加（数千～数万人規模）に伴う同時アクセス負荷に対して水平スケール可能な設計とする。※要確認：想定ユーザー数 | 高 |
| NFR-007 | 性能 | バッチ処理完了時間 | 夜間バッチにて全データソースのインクリメンタル更新を8時間以内に完了する。※要確認：データソースの更新頻度 | 中 |
| NFR-008 | セキュリティ | 通信暗号化 | すべての通信においてTLS 1.3以降を適用する。 | 高 |
| NFR-009 | 運用 | バックアップ・復旧 | データベースは日次でバックアップを取得し、障害時はRPO 24時間以内、RTO 4時間以内での復旧を目標とする。※要確認：許容されるデータ損失範囲 | 中 |
| NFR-010 | セキュリティ | AI利用における個人情報保護 | AIモデルへのクエリ送信時にユーザーの個人情報や非公開の研究内容が含まれないようマスキング処理を行う。※要確認：利用LLMのデータ学習ポリシー | 高 |
