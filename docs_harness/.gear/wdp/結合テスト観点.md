# 結合テスト観点

## テスト観点ツリー

### ITV-L1-001 DB結合

#### ITV-L2-001 SQL結合・検索 (論文検索・管理 / マイライブラリ管理)

- **ITV-L3-001 Research Corpusデータの検索・取得検証** [高]
  - research_papersテーブルからキーワード・発行年による検索結果が仕様通り取得できること
  - 検索対象レコードが存在しない場合に空配列が返却されること
  - 詳細取得時に指定されたIDの論文データが正確に取得できること

#### ITV-L2-002 トランザクション制御 (マイライブラリ管理)

- **ITV-L3-002 ユーザーライブラリ保存処理の整合性** [高]
  - user_librariesテーブルへの保存時にトランザクションが正常にコミットされること
  - 重複保存制限に違反した場合にロールバックされ、データが保護されること
  - ユーザーIDと論文IDの整合性が保持されていること

### ITV-L1-002 API結合

#### ITV-L2-003 HTTPメソッド別動作 (ユーザーログイン / 外部データ収集実行)

- **ITV-L3-003 認証および管理操作の実行結果検証** [高]
  - POST /api/auth/login が正常な資格情報でアクセストークンを返却すること
  - POST /api/admin/sources が正常に外部データ収集バッチをトリガーすること
  - 権限不足時に管理者用APIが適切に拒否されること

#### ITV-L2-004 異常パラメータ (論文検索 / AI論文要約)

- **ITV-L3-004 バリデーションエラー時の応答検証** [中]
  - 不正な論文ID指定時に404エラーまたは正しいエラーコードが返却されること
  - 検索条件が未定義または形式不正な場合に適切にバリデーションエラーとなること
  - 必須項目欠落時にHTTPステータスが適切に制御されていること

#### ITV-L2-006 認証連携 (全APIエンドポイント)

- **ITV-L3-006 認証失敗時の挙動検証** [高]
  - 未認証状態で認証が必要なAPIを呼び出した際に401 Unauthorizedが返却されること
  - 無効なアクセストークンを指定した場合に401 Unauthorizedが返却されること
  - 期限切れのアクセストークンを指定した場合に401 Unauthorizedが返却されること

#### ITV-L2-007 認可連携 (全APIエンドポイント)

- **ITV-L3-007 権限制御の検証** [高]
  - 一般ユーザーがadmin権限を必要とするAPIを呼び出した際に403 Forbiddenが返却されること
  - トークンの権限スコープが不足している場合に適切なエラーコードが返却されること
  - RBACに基づき、ユーザーの権限外の操作がブロックされること

#### ITV-L2-008 共通エラーハンドリング (全APIエンドポイント)

- **ITV-L3-008 システムエラー・通信制御の検証** [中]
  - バックエンド処理で予期せぬ例外が発生した際に500 Internal Server Errorが返却されること
  - 処理がタイムアウトした場合に適切なエラーレスポンスが返却されること
  - レート制限（Rate Limit）を超過したアクセスに対して429 Too Many Requestsが返却されること

### ITV-L1-003 画面⇄サーバ結合

#### ITV-L2-005 画面操作→結果反映 (論文検索 / トレンド分析)

- **ITV-L3-005 検索条件・分析結果の画面表示整合性** [高]
  - フロントエンドで指定した発行年・キーワードがサーバ側の検索結果に反映されること
  - トレンド分析データがグラフとして表示用に正しくマッピングされること
  - AI論文要約結果が詳細画面上で正しくレンダリングされること
